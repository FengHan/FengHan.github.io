认识MySQL

关系与非关系型数据库

关系型数据库

  • mysql
  • oracle
  • sql server
  • postgresql

非关系型数据库

  • hadoop
  • mongoDB
  • redis
  • Cassandra

区别：

是否使用SQL语句作为数据操作的方法和方式。

所以关系型数据库又叫做SQL数据库。

为什么学习MySQL

  • 基于GPL协议
  • Mysql 最流行 ，开源，但不是最先进的。
  • 互联网企业中占绝对主流地位

开发者学习MySQL：

  • 学习数据类型的正确使用
  • 学习如何利用索引优化查询效率
  • 了解分区，读写分离特性
  • 了解数据库事务相关特性

DBA学习MySQL：

  • 学习MySQL体系结构知识
  • 学习日常运维，性能调优，故障排查
  • 虚席数据层架构设计，schema优化知识

  • 企业版
  • 集群版
  • 社区版 （免费）

    Windows上安装

    安装包：300M+

Linux上安装

Ubuntu上安装MySQL

apt-cache search mysql-server
apt-get install mysql-server-5.6
#查看mysql进程
ps -ef|grep mysqld
#查看mysql状态：版本、运行时间、连接方式、查询统计
sudo /etc/init.d/mysql status
#msyql启动、停止、重启
sudo /etc/init.d/mysql start|stop|restart

连接使用MySQL

  • 应用程序驱动
  • MySQL命令行

  • GUI工具

    #查看mysql-client版本 mysql -V

命令行连接MySQL

  • Socket连接(本地连接)

      mysql -S/var/run/mysqld/mysqld.sock -uhomestead -p

  • TCP/IP连接（远程连接）

       mysql -h127.0.0.1 -P3306 -uhomestead -p

本地连接只能在MySQL服务器上创建，常作为MySQL状态检查，或程序和MySQL部署在一台机器上。

Ctrl + r : 搜索之前敲过的命令

两个重要的命令： status, show processlist

status

关注的信息：

  • Server version
  • characterset: utf8 (Server, Db, Client, Conn)
  • Uptime

    show processlist

help select;

#查看socket文件位置
show global variables like 'socket'
  • socket权限为777
  • 不要把密码直接输入在命令行里面，存在安全风险。

history -c 清楚命令行历史

认识DBA

DBA是最后一道屏障，守门员，是业务稳定的基石。

监控线上，保障业务的正常运行。

做好实时性能监控,一旦有长的SQL语句，就会立刻接到报警，然后进行调优。

小公司

  • 没有专职DBA
  • 但肯定有懂数据库的人

大公司：

  • 专职DBA
  • 数据存储技术专家

互联网业务： 高性能、高可用、可扩展、安全性。

没有DBA就没有稳定的数据库，请求变慢，数据丢失，安全问题，用户投诉，无穷无尽。

DBA要做哪些工作？

  1. 基础运维
    • 安全部署
    • 监控
    • 故障处理
  2. 安全运维工作
    • 数据备份与恢复
    • 安全访问，安全漏洞
    • 审计
  3. 性能调优工作
    • 数据库优化
    • 容量评估、软硬件升级
  4. 开发支持工作
    • 存储方案制定(哪些数据放在memcache，减少对数据库的访问)
    • 数据库设计
    • 数据库变更 SQL Review （审核业务SQL，全表扫描的SQL会把数据库跑死）
  5. 流程与培训
    • 数据库开发规范
    • 运维流程与标准化
    • 业务培训

练习： mysql要做的：

分析MySQL慢日志请求，优化表索引

DBA需要的技能

专业技能： - 数据库原理 - Linux与Shell - 计算机体系结构 - 网络原理 - 数据库系统与操作 - 服务器硬件 - 业务架构设计 软技能：

- 责任心、执行力
- 坚韧，抗压（半夜、凌晨工作）
- 学习与沟通能力
- 正直，诚信
- 耐心，注重细节
- 分析能力
- 团队协作
-

DBA如何发展？

  • 业务架构师（数据存储有深入研究，处理不同业务场景与需求的数据存储）
  • 运维专家（解决mysql的任何问题，帮助数据库更稳定更高效的运行）
  • 数据库研发（改进数据库系统或开发自己的数据库）
