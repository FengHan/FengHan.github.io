MySQL主从复制

Sep 10th, 2015 10:44 am

前言

之前写过一篇MySQL的主从配置，今天这篇文章涉主要涉及理论。

什么是主从复制

  • 数据拷贝
  • 准实时
  • 源主节点； 目的-从节点

主从复制的原理

复制格式

  • SRB: Statement Based Replication (原生的语句)
  • PBR: Row （5.7.7变为默认格式）
  • MBR: Mixed

查看复制格式 show global variables like ‘binlog_format’;

配置文件 [mysqld] binlog_format = ROW

主从复制的用途

  • 试试灾备，用于故障切换（高可用的切换）
  • 读写分离，提供查询服务
  • 备份，避免影响业务

MySQL主从复制的一些模式

  1. 一主一从
  2. 主主复制
  3. 一主多从
  4. 多主依从 （mysql5.7开始支持）
  5. 联级复制

主从复制的搭建

主从部署的必要条件：

  • 主从开启binlog日志（设置log-bin参数）
  • 主从server-id不同
  • 从库服务器能连通主库

主从部署步骤

  • 备份还原(mysqldump )
  • 授权 （grant replicaton slave on .）
  • 配置复制，并启动 (change master to)

  • 查看主从复制信息 (show slave status\G)

    主从复制的问题

  • 主库宕机，数据可能丢失

  • 从库只有一个sql thread, 主库写压力大，复制很可能延时。

MySQL半同步复制

MySQL并行复制

  • 社区版5.6新增
  • 并行是指从库多线程，

  • 库级别并应用binlog，同一个数据库更新还是串行的（5.7版并行复制基于事务组）

    set global slave_parallel_workers = 10
